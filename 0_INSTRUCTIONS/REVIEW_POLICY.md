# PYLLM_REVIEW_POLICY.md

이 파일은 학습용 코드 리뷰와 수정 제안의 기준을 정리하는 문서다.

## 핵심 원칙

- 사용자가 공부 완료한 파일들의 구조와 형태는 가능한 한 존중한다.
- 그 구조가 최신 코딩 방식과 조금 다르더라도, 바로 뜯어고치지 않는다.
- 먼저 유지 가능한 형태로 설명하고, 필요하면 "대안"으로 제안한다.

## 리뷰 시 행동 원칙

- "지금 방식도 학습용으로는 충분히 의미 있다"와 "실무에서는 이런 대안도 있다"를 구분해서 설명한다.
- 기존 구조를 바로 없애기보다, 왜 바꾸고 싶은지 이유를 먼저 분명히 한다.
- 학습 흐름을 끊는 대규모 리팩터링은 사용자가 원할 때만 한다.

## 제안 방식

- 기본:
  - 기존 패턴 유지
- 개선안:
  - 별도 제안으로 설명
- 사용자가 승인하면:
  - 그때 구조 변경 진행

## 특히 주의할 것

- 익숙한 네이밍 패턴을 함부로 바꾸지 않는다.
- 학습 목적상 의도적으로 풀어쓴 코드를 너무 빠르게 축약하지 않는다.
- "더 짧다"보다 "더 이해하기 쉽다"를 우선 평가한다.
