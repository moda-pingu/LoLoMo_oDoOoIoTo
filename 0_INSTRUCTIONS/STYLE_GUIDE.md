# PYLLM_STYLE_GUIDE.md

이 파일은 사용자가 익숙하게 느끼는 코드 스타일을 정리하는 문서다.

## 현재 선호 스타일

- 학습용 파일은 주석이 많고 설명형이어야 한다.
- 코드 블록은 단계별 번호와 섹션 주석으로 나누는 방식을 선호한다.
- 함수명, 노드명, 라우트명은 역할이 드러나게 짓는 편을 선호한다.
- `NODE_*`, `ROUTE_*`, `MY_*` 같은 이름 패턴에 익숙하다.
- `MY_` 접두사는 라이브러리에서 가져온 것이 아니라 내가 직접 만들고 이름 붙인 객체라는 뜻으로 써서, 구조를 볼 때 사용자 정의 객체와 라이브러리 제공 요소를 바로 구분할 수 있게 한다.
- `LangGraph` 예제는 `State`, `NODE`, `ROUTE`, `build_graph`, `run_chatbot` 구조로 보는 것이 익숙하다.
- 디버깅 함수는 별도로 두고, 각 step의 상태와 메시지를 출력하는 구조를 선호한다.

## 변수명 / 함수명 경향

- 설명적인 이름을 선호한다.
- 학습용 예제에서는 짧은 축약보다 의미가 드러나는 이름을 더 선호한다.
- 단, 이미 익숙한 기존 예제 스타일이 있으면 그 스타일을 우선 따라간다.

## 주석 스타일

- "왜 이 코드가 필요한지"를 설명하는 주석 선호
- 새 개념은 쉬운 말로 풀어주는 주석 선호
- 예:
  - `with_structured_output()`
  - `BaseModel`
  - `Field(default_factory=list)`
  - `ToolMessage`

## 구조 스타일

- 너무 감춰진 추상화보다 눈으로 따라가기 쉬운 순차 구조 선호
- 노드별 역할이 분명한 구성을 선호
- 한 파일 안에서 전체 흐름을 먼저 보고, 세부 함수로 내려가는 형태를 선호

## 앞으로의 작성 원칙

- 새 파일 작성 시 이 문서의 스타일을 최대한 따른다.
- 더 좋은 구조가 있어도 사용자가 익숙한 구조와 충돌하면 바로 바꾸지 말고 제안한다.
