# PYLLM_LEARNING_PROFILE.md

이 파일은 사용자의 공부 방식을 누적 기록하는 문서다.

## 현재까지 파악된 공부 성향

- 목적은 "코드가 돌아가는 것"보다 "코드 구조를 이해하고 나중에 직접 수정할 수 있는 것"에 더 가깝다.
- 새 클래스, 새 함수, 새 구조, 새 라이브러리 사용이 나오면 쉬운 설명이 필요하다.
- 기존에 익숙한 패턴과 비슷한 구조를 선호한다.
- 주석이 충분히 달린 학습용 코드를 선호한다.
- 한 번에 너무 추상적인 리팩터링보다, 눈으로 따라가기 쉬운 구조를 선호한다.
- `LangGraph`, `Tool`, `Structured Output`, `MemorySaver`, `Pydantic BaseModel` 같은 개념은 설명을 붙여가며 익히는 중이다.

## 코드 작성 시 반영할 것

- 새 개념이 나오면 쉬운 말로 설명 주석 추가
- 이미 익숙한 파일 스타일 우선 사용
- 구조를 나누더라도 지나치게 감춘 헬퍼 함수 남발은 피하기
- 디버깅 함수, 중간 출력, 상태 확인 코드를 학습용으로 남기는 편을 우선 고려

## 이후 누적할 내용

- 사용자가 특히 이해하기 어려워했던 구조
- 사용자가 선호한다고 확인한 설명 방식
- 자주 쓰는 예제 패턴
- 반복적으로 요청한 네이밍/주석 스타일
